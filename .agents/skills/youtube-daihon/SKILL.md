# YouTube台本作成ワークフロー

参考YouTube動画のURLを起点に、対話形式で台本を完成させるスキル。**必ず以下のステップを順番通りに実行**してください。各ステップ完了後の確認質問・了承待ちを省略してはいけません。

## 全体ルール

- 話者は「AI駆動開発ラボのりょう」、一人称は **"僕"**
- 各ステップで指示されている確認質問は **必ず実行し、ユーザーの了承を待つ**
- 確認質問の前にステップを進めない（連続実行しない）
- 修正指示があれば、該当ステップを再実行する
- ステップ間で出力した内容は会話履歴に残し、後続ステップで参照する
- `**` (アスタリスク) は本編で使用しない
- 1文ごとに改行を入れる（ただし短い文を3連続以上並べない。重要注意点 J を参照）
- 視聴者は AI 駆動開発の中級者〜上級者が集まりやすい。**業界の常識・既出の内容は解説せず、すぐ本題に入る**（重要注意点 H）

## 参考台本（YouTube運営代行の添削でOKをもらった完成版）

`src/4/script.md`・`src/5/script.md`・`src/6/script.md` は、いずれも YouTube 運営代行の台本添削で **OK をもらった最新の承認済み完成台本**。**OP・本編・中間CTA・ED作成時（STEP 11, 12, 13, 14）は必ず Read ツールで3ファイルすべてを読んでスタイルに揃える**。

- `src/4/script.md` — 「神機能5選」型。機能紹介ごとに自己体験・感情を挟む手本。中間CTAは「Skills 紹介の流れ → Claude Code と組み合わせる疑問」から自然に入る。EDは「さらに、Claude をもっと極めていくと〜」のビジョン展開型
- `src/5/script.md` — 「30分でアプリを作る」型。本編で「GitHub・デプロイ」を伏線として張り、中間CTAで「ここで絶対に押さえておきたいのが、さっき紹介した GitHub とデプロイ」と地続きに回収する手本。EDは「さらに、Claude Code とそれに付随する技術を極めれば〜」のビジョン展開型
- `src/6/script.md` — 「言語おすすめ3選」型。**OPの締めが「〜できるようになるためにも、ぜひ最後までご覧ください！」と理想の未来＋締めの最良の手本**。中間CTAは「学ぼうと思っても何から手をつければ？」という視聴者の迷いから入る。EDは「この3つの言語を極めて、さらに AI まで使いこなせるようになれば〜」のビジョン展開型

旧サンプル `src/2.md` も承認済み台本として残っているが、**今回の最新OK台本は上記3本**なので、作成時はまず3本を最優先で踏襲する。

### 採用すべきスタイル要素

1. **セクション区切り**
   - `━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━` で本編のトピック単位を区切る
   - セクション見出しは `■ 本編 1: 〜` / `■ 本編 2: 〜` の形式
   - 構造: `■ OP` → `■ 本編 1〜N` → 必要に応じて `■ 中間 CTA` → 続きの本編 → `■ エンディング・CTA`

2. **OP の型**
   - 冒頭にタイトルコール（編集指示 `（実際に作ってる画面を見せる、編集で）` などを併記してよい）
   - 続けて `こんにちは!AI 駆動開発ラボのりょうです!`
   - 自己紹介スライド指示ブロックを `━━━━━━━━━━━━━━━━━` で囲む（**最新の承認台本に合わせた内容を使う**）
     ```
     【自己紹介スライド表示】
     - 株式会社 Vertex 代表
     - AI 駆動開発スクール「Happiness Chain AX」を運営
     - 受講生 累計 350 名以上の指導経験
     ```
   - その後にタイトル正式コール「今日は、『〜』をやっていきます!」
   - ベネフィット提示（OPで全部を語りすぎない）
   - **OPの締めは「この動画を見ることで得られる理想の未来」を提示してから締める**。`〜できるようになるためにも、ぜひ最後までご覧ください！` のように、視聴後に手に入る理想の状態を一言で描いてから締めくくる（src/6 の OP 末尾が手本）
     - 例: `AI で作ったアプリを、自分の手で自由自在にカスタマイズできるようになるためにも、ぜひ最後までご覧ください！`
     - 動画によっては `ぜひ一緒に〜していきましょう！`（src/4・src/5）でも可。どちらも「理想の未来 → 前向きな締め」の型を守る
     - **重要**: 「最後までご覧ください」は **全員対象の言い方**（`〜できるようになるためにも、ぜひ〜`）でのみ使う。「〜したい方は最後までご覧ください」のような選択肢提示は NG（後述の重要注意点 B を参照）

3. **編集・演出指示の埋め込み**
   - `(スライドを表示)` / `(以下の説明でスライドを表示)` — スライド差し込み
   - `(ここから画面を見せながら操作)` — 画面録画パート開始
   - `(3 分ほど経ちました)` / `(1 分ほど経ちました)` — 時間経過カット
   - `(NetFlix)(モザイク)` — モザイク処理指示
   - 必要なタイミングで括弧書きで編集者向け指示を入れる

4. **プロンプト・指示文の見せ方**
   - 視聴者に提示するコマンド・プロンプト例は `━━━━━━━━━━━━━━━━━` で囲み、上部に `【プロンプト】` を置く

5. **文体・話法の特徴**
   - 視聴者代弁: 「『〜』そう思いましたよね?」「『〜』って思ってた方、安心しましたよね?」
   - **自己体験を良い頻度で入れる**: 各トピック（本編セクション）に最低1回は、自分が実際に使った・学んだときの体験を入れる。「僕自身、これを初めて使った時、え、めっちゃ楽じゃんで、感激しました!」「僕も最近、めちゃくちゃ使っています。」「僕自身、React・Next.js を極めたことで、エンジニアに転職できました」「僕自身、Go を初めて勉強したときは、本当に苦労しました」のように、実体験を証拠・説得材料として散りばめる
   - **感情を良い頻度で入れる**: 機能やノウハウを紹介した直後に、自分の感情をそのまま言葉にする。「『え、もうやってくれてたの?』って、初めて見たとき本当に声が出ました。」「これ、何回使っても慣れないんですよ。毎回ちょっと感動する。」「いや、もうコスパ良すぎて怖いくらいです」「マジでやばくないですか?」「便利すぎて笑いが出てきますよね」など
   - **体験・感情の入れすぎにも注意**: 毎文に入れると自分の話ばかりになって逆効果。「事実・ノウハウ → 自己体験 → 感情」のリズムで、トピックごとに自然な頻度で配置する
   - 数字で説得: 「累計500名以上」「月3,000円」「最低でも丸4日はかかります。それが、たった1時間。」
   - 常識破壊: 「黒い画面でカチャカチャやるんでしょ? って思ってた方、安心しましたよね?」
   - **上級者に刺さる踏み込み**: 各トピックに「よくやるミス」「巷ではこう言われているが半分正解・半分間違い」を最低1つ入れる（重要注意点 I）
   - **文のリズム**: 短い文を3連続以上並べない。接続詞・助詞でつないで意味のまとまりを作り、長い文と短い文で緩急をつける（重要注意点 J）
   - 親しみある語尾: 「〜なんですよね」「〜なんです」「〜ですよ」を多用
   - 半角スペース: 英数字と日本語の間、半角記号の前後に半角スペース（例: `Claude Code を使う` / `月額 20 ドル前後`）

6. **中間 CTA の構成（本編中盤に1ブロック）**
   - **「中間CTAに入った」と思わせない自然な入りを最優先する**。直前の本編内容から地続きで、視聴者の中に自然に湧き上がる「困りごと・迷い・次に知りたいこと」を起点に橋渡しする。CTA の合図（「ここで一つお知らせです」等）を冒頭に置かない
     - src/4: 本編3（Skills）の流れから「さらに、Claude Code で Skills を使うと〜。ただ、どう組み合わせるの? という疑問、ありますよね」と地続きに入る
     - src/5: 本編で「GitHub・デプロイ」を伏線として張っておき、「そして、これから実装に入るのですが、今ここで絶対に押さえておきたいのが、さっき紹介した GitHub とデプロイです」と回収して入る
     - src/6: 「学ぼうと思っても、何から手をつければいいの? どの教材で? って迷っちゃいますよね」という視聴者の迷いから入る
   - 特典名を提示 → LINE 友だち追加 → 5文字以内のキーワード送信 → 受け取り
   - LINE登録の誘導は「**ここで全員概要欄のリンクから**僕の公式 LINE をお友だち追加し」という全員対象の言い方を使う
   - 例: 「ここで全員概要欄のリンクから僕の公式 LINE をお友だち追加し、『〇〇』とメッセージ！すると、すぐにこんな感じで届きますので、〜」

7. **EDの構成**

   最優先は **「EDに入った」と思わせない自然な入り**。最新の承認台本3本（src/4・5・6）は、いずれも本編最終トピックの結びから「**さらに、〜を極めると/極めれば…**」と地続きにビジョンへ展開する型で統一されている。これを基本形とする。

   **基本形「ビジョン展開」型**（src/4・src/5・src/6 の ED が手本）
   - 本編で身につけた内容を受けて「さらに、〜を極めると、できることの幅が一気に広がります」と地続きで前向きに展開する
     - src/4: 「さらに、Claude をもっと極めていくと、できることの幅が一気に広がっていきます。」
     - src/5: 「さらに、Claude Code とそれに付随する技術を極めれば、〜全部自分で作れるようになります。」
     - src/6: 「この3つの言語を極めて、さらに AI まで使いこなせるようになれば、〜思いどおりにできるようになります。」
   - 動画テーマを超えた、人生・キャリア・働き方レベルの大きな可能性へ視野を広げる
   - 個人の具体的な実績を証拠として添える（「技術顧問や複数企業の開発支援が増えた」「旅先でもパソコン一台で仕事ができる」「企業から難しい大規模開発を任せてもらえる」など）
   - 「あなたにもそうなってほしい。」「次は、あなたの番です。」と感情的に訴えかける
   - LINE特典・ウェビナーを「まだの方は、必ず概要欄のリンクからお友だちになって、全部受け取ってくださいね！！」で訴求する

   **別形「落とし穴→特典」型**（旧 src/2.md 系。動画に合えば選んでよい）
   - 本編最終トピックの結びから「ただ一つだけ大きな落とし穴があって〜」と地続きで入る
   - 本編で扱った技術の限界・前提条件を切り出し、「だからこそ特典が必要」と論理接続する

   **共通の締め（どちらの型でも使う）**
   - 特典訴求の後に、その動画固有の前向きな一言を入れる（「一緒に AI で人生を変えていきましょう！」「AI での開発をもっと極めていきましょう！」など）
   - like/subscribe は「この動画が「役に立った!」と思っていただけたら、高評価とチャンネル登録をぜひお願いします。」を使う（「最後までご視聴いただきありがとうございました。」は使わなくてよい）
   - 「また、次の動画でお会いしましょう。バイバイ。」で締める

8. **トピックコールに惹きを作る**
   - 単に「本編1: 〇〇とは?」ではなく、内容が想像できるタイトルにする
   - 例: 「本編5: 計画書の修正」のように、視聴者がそのSTEPで何をするか分かる命名

9. **やってはいけない（最新の承認台本3本で守られている禁則）**
   - 冒頭での目次列挙なし
   - 後半でのまとめ反復なし
   - `**` 記号は使わない（強調はカギカッコや読点で表現）
   - 1文ごとに改行を入れて読みやすくする
   - 業界の常識・既出テーマの前提解説を長々とやらない（重要注意点 H）
   - 短い文の連打で"ぶつ切り"にしない（重要注意点 J）

## 過去のフィードバックから学んだ重要注意点

過去の台本レビューで指摘されたポイント。**OP・本編・中間CTA・ED作成時は必ず守る**。

### A. 「この動画を見ることで」系の表現を使わない

視聴者は既に動画を見ている前提。「この動画を見ることで〇〇になります」という言い方は冗長で、視聴者に対して説明的すぎる。

- ❌ `この動画を見ることで、〇〇ができるようになります`
- ⭕ `〇〇ができるようになります` / `〇〇を、惜しみなくお伝えしていきます` / `一緒に手を動かしながら、〇〇を作ってみましょう`

ベネフィット提示の表現は、視聴後の状態や行動を直接描写する形に置き換える。

### B. 視聴者に選択肢を委ねる言い方を絶対にしない（OP・CTA共通）

「〜したい人は」「〜に興味がある人は」「〜の方はぜひ最後までご覧ください」という選択肢提示は禁止。動画を最後まで見ている時点で **全員が対象**。「自分は対象外」と思わせて離脱させる原因になる。

- ❌ `Webエンジニアという職種に興味がある人は、ぜひ受け取ってください`
- ❌ `少しでもWebエンジニアという働き方に興味がある方は〜`
- ❌ `〇〇したいと思っている人に向けた特別ウェビナー`
- ❌ `こういった方はぜひ最後までご覧ください`

- ⭕ `Webエンジニアという働き方で人生を切り拓くためにも、ぜひ概要欄のリンクから、必ず参加してくださいね`
- ⭕ `AI時代のWebエンジニアとしてキャリアを再構築するための特別無料ウェビナー`
- ⭕ `こういった未来を叶えるためにも、〇〇`
- ⭕ `AI で作ったアプリを、自分の手で自由自在にカスタマイズできるようになるためにも、ぜひ最後までご覧ください！`（**全員対象の「〜できるようになるためにも、ぜひ最後までご覧ください」は OK**。NG なのは「〜したい方は最後までご覧ください」のような選択肢提示の言い方）

判断基準：その文を読んだ視聴者の半分が「自分は対象じゃないな」と思う言い回しは NG。**全員が受け取る前提**で書く。

### C. 中間CTAをパターン化しない

毎回同じ流れ（「ところで、〜面倒ですよね → 実は便利な設定があります → LINEで配布」）にすると、視聴者に「またこのパターン来た」「結局LINE登録させてウェビナー見せたいだけでしょ」と営業臭く感じさせて離脱に繋がる。

- 中間CTAは **動画ごとに違う角度・違う流れ・違う「自然な受け取る理由」** で組み立てる
- 直前の本編内容から自然に湧き上がる「視聴者の困りごと・知りたいこと」を起点にする
- 特典の渡し方も、「便利ツール」「設定マニュアル」「テンプレ集」「事例集」「失敗回避ガイド」など、その動画固有の文脈に最も馴染むものを選ぶ
- 「LINE登録 → キーワード送信 → 受け取り」という構造自体は同じでよいが、**そこに至るまでの導入・理由付け・特典名** は毎回変える

最新の承認台本3本も、それぞれ違う角度で中間CTAを組んでいる。src/4 は「Skills を Claude Code と組み合わせる方法が知りたい」という疑問起点、src/5 は本編で張った「GitHub・デプロイ」の伏線を回収する起点、src/6 は「学びたいが何から手をつければ迷う」という起点。次の動画でこれらと同じ流れを使い回すのは NG。

### D. EDで「いかがだったでしょうか」「動画はここまで」系の言い方を使わない

「いかがだったでしょうか」「以上、〇〇でした」など、動画が終わることを匂わせる定型句を使うと、視聴者は「あ、もう終わりか」と感づいて離脱する。CTAやプレゼント訴求まで聞いてもらえない。

代わりに、**本編最終トピックの結びから地続きで** ED に入る。「すごいですよね、ただ一つだけ大きな落とし穴があって〜」のように、本編で語った技術・手法の **限界点・注意点・前提条件** を切り出して、そこから「だからこそ〇〇が必要」→「そのための特典を配布している」と論理接続する。

- ❌ `いかがだったでしょうか。今日は〇〇についてお話ししました。`
- ❌ `以上、〇〇の解説でした。`
- ⭕ `さらに、〇〇をもっと極めていくと、できることの幅が一気に広がっていきます。`（最新3本の基本形＝ビジョン展開型の入り）
- ⭕ `〇〇ができるんです。すごいですよね。ただ一つだけ大きな落とし穴があって〜`（落とし穴型の入り）

**最新の承認台本3本（src/4・5・6）は、いずれも「さらに、〜を極めると/極めれば…」と本編最終トピックから地続きにビジョンを展開する型で統一されている**。動画テーマを超えた大きな可能性→個人実績→「あなたにもそうなってほしい/次はあなたの番です」→特典CTAと繋ぐ。落とし穴型も選択肢として残すが、まずはこのビジョン展開型を基本形とする。どちらの型でも「いかがだったでしょうか」は NG。

### E. OPの締めは「理想の未来」を提示してから締める

OPの最後は、本編に入る前に **この動画を見ることで視聴者が手に入れる理想の未来（ベネフィットの到達点）を一言で描いてから締める**。「〇〇できるようになるためにも、ぜひ最後までご覧ください！」のように、視聴後の理想の状態 → 前向きな締めの順で組む（src/6 の OP 末尾が手本）。

- ⭕ `AI で作ったアプリを、自分の手で自由自在にカスタマイズできるようになるためにも、ぜひ最後までご覧ください！`
- ⭕ `是非一緒に手を動かしながら、アプリを作っていきましょう！`（理想の未来を本編全体で描くタイプの動画はこの締めでも可）
- 注意: 「最後までご覧ください」は **必ず全員対象の言い方**（重要注意点 B）で使う。「〜したい方は〜」は NG。

### F. 自己体験と感情を「良い頻度」で入れる

機能・ノウハウを淡々と説明するだけでなく、各トピックに **自分の実体験と、その時の感情** を散りばめる。「事実・ノウハウ → 自己体験 → 感情」のリズムを意識する。

- 自己体験の例: `僕自身、これを初めて使った時、え、めっちゃ楽じゃんで、感激しました！` / `僕自身、React・Next.js を極めたことで、エンジニアに転職できました` / `僕自身、Go を初めて勉強したときは、本当に苦労しました`
- 感情の例: `「え、もうやってくれてたの?」って、初めて見たとき本当に声が出ました。` / `これ、何回使っても慣れないんですよ。毎回ちょっと感動する。` / `便利すぎて笑いが出てきますよね`
- ただし毎文に入れると自分の話ばかりになり逆効果。トピックごとに自然な頻度で配置する。

### G. 中間CTA・EDは「入った」と気づかせない自然な入りにする

視聴者に「あ、CTA来た」「あ、もう終わりか」と感づかれると、そこで離脱する。**直前の本編内容から地続きで** 中間CTA・EDに入る（重要注意点 C・D とスタイル要素 6・7 を参照）。

- 中間CTA: 直前の本編から自然に湧き上がる「困りごと・迷い・次に知りたいこと」を起点に橋渡しする。CTA の合図を冒頭に置かない。本編であらかじめ伏線（例: GitHub・デプロイ）を張っておき回収するのも有効
- ED: 「さらに、〜を極めると…」と本編最終トピックの成果を受けて前向きに展開しながら入る

### H. 業界の常識・既出の内容は解説せず、すぐ本題に入る

このチャンネルには AI 駆動開発の中級者〜上級者が集まりやすい。業界ですでに周知の事実（AI コーディングツールの存在、AI で開発が速くなること、基本的な用語の定義など）や、自分が別の動画ですでに詳しく語ったテーマを一から説明し直すと、「その話は知ってる」と思われて即離脱される。

- 前提・定義・基礎の説明は **1〜2文で流す**。深掘りしない
- すでに別動画で語った内容は「詳しくは別の動画で話しているので、今日はその先の話をします」と一言で片付け、本題へ進む
- 本編は「第1位」「STEP 1」から、**その動画でしか聞けない独自のノウハウ**にすぐ入る

- ❌ `そもそも〇〇とは、〜というツールで、〜ができて、〜も便利で、〜という特徴があって…`（前提説明が5文以上続く）
- ⭕ `〇〇はもうみなさん使っていると思うので、今日はその一歩先、〜の話からいきます。`
- ⭕ `〇〇の基本的な使い方は別の動画で詳しく話しているので、今日は実務でつまずくポイントに絞ります。`

判断基準：その説明を聞いて上級者が「それは知ってる」と思うなら、削るか一言に圧縮する。

### I. 「よくやるミス」「巷の通説は半分正解」を本編に散りばめる

上級者が集まりやすいからこそ、**他では聞けない踏み込んだ話** が刺さる。各本編トピックに最低1つ、次のどちらかを入れる。

1. **よくやるミス**: `これ、みんな最初にやりがちなんですけど、実は逆効果なんですよ。`
2. **通説の是正（半分正解・半分間違い）**: `巷では「〜すればいい」ってよく言われますよね。あれ、半分は正解なんですけど、半分は間違いなんです。`

組み立ての型：**「巷ではこう言われている」→「半分は正解」→「でも、ここが抜けている」→「だから、正しくはこうする」**

- 頭ごなしの否定にはしない。一度受け止めてから、条件・前提を足して正す（「批判的になり過ぎない」ルールと両立させる）
- 自分が実際にそのミスをやらかした体験（重要注意点 F）と組み合わせると説得力が跳ね上がる
  - 例: `僕も最初は全部これでやってたんですけど、途中で完全に破綻しました。`
- 「みんなが知らない裏側」「実務で分かった落とし穴」も同じ役割を果たす

### J. 短い文章を連続で並べない

短文を3つ以上連打すると、箇条書きを読み上げているように聞こえ、テンポではなく "ぶつ切り" になる。**「1文ごとに改行」のルールはそのまま守るが、1文の中身は接続詞・助詞でつないで意味のまとまりを作る。**

- 短い文（20文字前後）を **3連続以上させない**。長い文と短い文を混ぜて緩急をつける
- 言い切りの短文は「効かせどころ」で1つだけ使う。連発すると効果が消える

❌ 悪い例（短文の連打）
```
これ、めちゃくちゃ便利です。
一瞬で終わります。
もう手作業はいりません。
本当に楽です。
```

⭕ 良い例（つないで緩急をつける）
```
これ、今まで手作業で30分かかっていたところが、たった一言で終わるんですよ。
もう手作業には戻れないです。
```

- 初見でも「なんかこの人凄そう！」と思われるように

**「過去のフィードバックから学んだ重要注意点」セクションの C・F・G・H・I・J を必ず守る**：
- C: 中間CTAをパターン化しない。3本の中間CTAと同じ流れを使い回さない。今回の本編内容に固有の「自然な受け取る理由」を毎回違う角度で組み立てる。営業臭くしない。
- F: 自己体験と感情を「良い頻度」で入れる。各トピックに最低1回は実体験＋感情を散りばめる（入れすぎは逆効果）。
- G: 中間CTAは「入った」と気づかせない自然な入りにする。直前の本編から地続きの困りごと・迷い・伏線回収から橋渡しする。
- H: 業界の常識・既出テーマは解説せず、すぐ本題に入る。前提・定義・基礎の説明は1〜2文で流し、その動画でしか聞けない独自ノウハウに文字数を割く。
- I: 各トピックに最低1つ、「よくやるミス」または「巷ではこう言われているけど半分正解・半分間違い」を散りばめる。上級者が「他では聞けない」と感じる踏み込みを作る。
- J: 短い文を3連続以上並べない。接続詞・助詞でつないで意味のまとまりを作り、長い文と短い文で緩急をつける。

### 考慮すべきポイント

- 書き方は基本的に結論から始まる **PREP法** の型を活用
- 視聴者の代弁や常識の破壊から話を展開できると GOOD
- 文章を長くし過ぎずテンポ良く。演者が読みやすく、聞いてる側の聞き心地を意識
- 自分の強みを理解し、競合の上位互換となるような情報・ノウハウ・実績を散りばめる
- 自分ならではの活かせる素材も合わせて台本作成できると良い
  - 例：実績者のビフォーアフター/声のキャプチャ、独自の教材など
- ナンバリング整理を活用し、視聴者に伝わりやすいように情報を与える
- 各トピックの締めは視聴者が行動しやすいように伴走感などメッセージ性を加える
- 将来的には編集をイメージしながら書ける
- トピックコールに惹きを最大限に作ること。例：STEP1なら「学習時間は〇〇時間でOK！〇〇だけ学べ！」など、具体的なSTEP名にする。STEPを聞いて、ある程度何をするのかわかるようにする
- 本編はOP後「第〇位」や「STEP1」、「1つ目」など内容にすぐ入る。前提知識などをダラダラ喋らない（重要注意点 H）
- 視聴者は中級者〜上級者。**「よくやるミス」「巷ではこう言われているけど、実はこうするのが良い（半分正解・半分間違い）」を本編全体に散りばめる**（重要注意点 I）
- 短い文を3つ以上連続させない。接続詞・助詞でつないで、長短の緩急をつける（重要注意点 J）
- 視聴者の代弁の時、「〜と思ったかもしれませんね？で」ではなく、「～、そう思ったかもしれませんね？」という感じで、一回視聴者の気持ちをそのまま記載して、そこは本人が演じて喋りやすくする
  - 例：×「『そう簡単に上手くいく訳ないでしょ！』って疑いましたよね？」
  - 例：◯「『そう簡単に上手くいく訳ないでしょ！』そう疑いましたよね？」
- **接続詞の使い方**：前後の文脈の流れを作るために「しかし」「ただ」「でも」「そして」「あと、」「で、」「また」「さらに」「しかも」というような接続詞を入れることを必ず意識する。ただし、直近で同じ接続詞を使わない

### やってはいけないこと

- EDまで書き出してしまうこと（本編のみが作成対象）
- 言っていることが企画・タイトルの趣旨から逸脱する
- 伝えていることが視聴者にとって当たり前だったり惹きが全くない
- 同じ内容を何度も繰り返し展開する。序盤の目次や後半のまとめ等も不要
- 直近で同じ言葉、表現、文末、接続詞を何度も使う。聞いていて耳障りになりやすい
- 情報が偏ったり、批判的になり過ぎたり、実績や自分の話ばかりになる
- 伝えていることがアレもコレもと断片的になり過ぎて、繋がりがなく、意図が伝わらない
- 冒頭にチャンネル登録の誘導やLINE誘導などして、有益な情報を与えてない段階でCTAを設ける
- 伝えたいことが伝わりずらい
- 読みにくい、テンポが悪い
- てにをはのミス、不自然な言葉遣いや文末
- 同じ内容や同じような言い回しの繰り返し
- 間延びしている、煽り過ぎ、自分の話ばかり
- CTAが唐突、不自然な流れ
- `**` をつける
- 1文を必ず改行しない
- 業界の常識・既出テーマを一から丁寧に解説してしまう（上級者に「知ってる話だ」と離脱される）
- 短い文を3つ以上連打して、"ぶつ切り"で箇条書きの読み上げのようにしてしまう
- 通説の是正を頭ごなしの否定にする（一度受け止めてから、条件を足して正す）

本編出力後、以下を**そのまま聞く**：

> 「本編はこれで大丈夫ですか？次はEDを作成して良いでしょうか？」

了承されたら STEP 13 へ進む。

## STEP 13: ED作成

EDを**500文字**で書き出す。

**書き出す前に必ず `src/4/script.md`（ED: 行404〜422）・`src/5/script.md`（ED: 行753〜769）・`src/6/script.md`（ED: 行375〜393）を Read して、EDのパターンを確認してから書き出す。** 特に以下を真似る：

- セクション見出し `■ エンディング・CTA`
- 「採用すべきスタイル要素 7」の基本形＝ビジョン展開型（「さらに、〜を極めると…」と本編最終トピックから地続きに入る）を使う。動画に合えば落とし穴型でもよい
- 個人実績 →「あなたにもそうなってほしい/次はあなたの番です」→ 特典CTA の流れ
- like/subscribe は「この動画が「役に立った!」と思っていただけたら、高評価とチャンネル登録をぜひお願いします。」を使う
- 「また、次の動画でお会いしましょう。バイバイ。」で締める

**「過去のフィードバックから学んだ重要注意点」セクションの B・D・G を必ず守る**：
- B: 「〜に興味がある方は」「〜したい人に向けた」など視聴者に選択肢を委ねる言い方を絶対にしない。動画を最後まで見ている時点で全員対象。「〇〇のためにも、ぜひ全員に〜」というスタンスで書く
- D: 「いかがだったでしょうか」「以上、〇〇でした」など動画終了を匂わせる定型句は使わない
- G: EDは「入った」と気づかせない自然な入りにする。本編最終トピックの結びから「さらに、〜を極めると…」と地続きで ED に入る

### 考慮すべきポイント

- CTAやプレゼント訴求は動画毎に変え、視聴者が行動する理由や感情の動きを付ける
- 視聴者目線に立ち、「なぜ、わざわざLINE登録しなきゃいけないのか？」「なぜ、今その特典を受け取らなきゃいけないのか？」を考える。そして一番、視聴者が喜ぶもの。手に取って今すぐ使えるものを特典として訴求する
- プレゼント訴求の後は離脱されやすいので、無料公開の理由などの話の展開は入れず「プレゼントは特別であること」「LINE登録後のキーワード」「チャンネル登録、いいね訴求」「次回の動画でお会いしましょう」と入れ、**250文字ほどで早めに終える**

### やってはいけないこと

- CTAやプレゼント訴求をテンプレ化する
- CTAのキーワードを覚えにくいものにする。**5文字以上にしない**
- 前の話の文脈を無視して、繋がりがないものとする
- 動画の本編内容と、最後に渡すプレゼント（特典）のつながりを不自然にする
- 視聴者が「今すぐ受け取らなきゃ」と思うような論理的な誘導（例：CTAの指示部分に「〇〇について、さらに深く知りたい方のために〜」等）ができていない

ED出力後、以下を**そのまま聞く**：

> 「これで大丈夫ですか？LINE登録への繋ぎ方や渡す特典内容など、動画の本編内容とつながりが不自然になっていないか、CTAのキーワードを覚えにくいものや5文字以上にしていないか、視聴者が「今すぐ受け取らなきゃ」と思うような論理的な誘導（例としてCTAの指示部分に、「〇〇について、さらに深く知りたい方のために〜」等）ができているか、再度確認してください。」

- 修正指示があれば STEP 13 を再実行する。
- 了承された場合は STEP 14 へ進む。

## STEP 14: 統合台本の出力

ここまでの **OP、本編、ED** をまとめて 1 つの台本として書き出す。

- **本編は省略せず、全て書き出す**。
- 構成・体裁は `src/4/script.md`・`src/5/script.md`・`src/6/script.md` を踏襲する：
  - 各セクションの上下を `━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━` で区切る
  - 見出しは `■ OP` / `■ 本編 N: 〜` / `■ 中間 CTA`（あれば）/ `■ エンディング・CTA`
- 出力順：
  1. `■ OP`（300文字）
  2. `■ 本編 1〜N`（合計7000文字、省略なし）
  3. `■ 中間 CTA`（本編中盤に組み込んだ場合）
  4. `■ エンディング・CTA`（500文字）
- 完成版は会話に出力すると同時に、**`src/P/script.md` に Write ツールで保存**する（P はセッション開始時に決定済みのプロジェクト番号）。
- 保存後、ユーザーに保存先パスを伝えて完了報告する。
