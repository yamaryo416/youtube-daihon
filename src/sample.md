■ 本編 1:Claude Code とは?

実際に Web サイトを作り始める前に、まずは Claude Code とは何か、簡単に解説していきます。

Claude Code は、「エージェント型 AI」と呼ばれています。

「エージェント型って何?」と思いますよね。

ひとことで言うと、自分で考えて、実際に作業までしてくれる AI です。

(スライドを表示)

たとえば、取引先からのメールに返信したいとしましょう。

ChatGPT や Gemini の場合、過去のやり取りをコピーして、AI に文面を考えてもらい、それをメールに戻して送信。

この一連の作業を、全部自分でやる必要があります。

つまり AI がやってくれるのは『回答を返すこと』だけ。これが『チャット型 AI』です。

(スライドを表示)
一方で、Claude Code のようなエージェント型 AI なら、

「この相手に、いい感じに返信しておいて」

これだけ。

あとは過去のやり取りの参照から、文面の作成、送信まで、全部自動でやってくれるんです。

僕が初めて使ったとき、「え、これで送信できたの?」って衝撃を受けました、、

チャット型は「考えてくれる」、エージェント型は「考えて、実行までしてくれる」。 ここが決定的な違いなんです。

(スライドを表示)

これは Web サイトを作るときも、まったく同じです。

普通なら、自分でファイルを一つ一つ作って、コードを書いて、エラーが出たら直して、、、という作業が必要です。

でも、Claude Code なら「こんな Web サイトを作って」と伝えるだけで、ファイル作成からコードの記述まで、全部自動。

そして、出来上がったものに「ここをもっとこうして」と指示を繰り返していくことで、プロ級の Web サイトを作り上げることができるんです。

そんな Claude Code を使うために、まずは準備をしていきましょう。

■ 本編 4:要件定義
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ここで一つ、忘れずにやっておきたいのが、「New Session」ボタンをもう一度クリックして、新しい会話を始めることです。

なぜなら、Claude Code は今までの会話を引きずって動いてしまうから。

(以下の説明でスライドを表示)

例えば、「カレーの作り方」を聞いた後に、「資料を作って」とお願いすると、AI は「カレーの話の続き」だと思って、カレーをテーマにした資料を作り始めてしまうんです。

テーマが変わるときは、新しい Session で始めるのが鉄則です。

今回作るのは、「カフェの Web サイト」です。

ここで、僕が実務で必ずやっている、めちゃくちゃ重要なコツをお伝えします。

(以下の説明でスライドを表示)

初心者の方がやりがちなのが、「おしゃれなカフェの Web サイトを作って!」と、いきなり丸投げしてしまうこと。

これ、一番やっちゃいけないパターンなんです。

いきなり作らせると、AI は自分の解釈で作り始めてしまうので、「思ってたのと違う…」となりがち。

そして、ここでもう一つ知っておいてほしいのが、「使用制限」と「トークン」の存在です。
(以下の説明でスライドを表示)
Pro プランは月額 3,000 円で使い放題に見えますが、実は使える量に上限が決まっているんです。
そして、AI とのやり取りの『文量』のことを「トークン」と呼びます。指示や回答のたびに、このトークンが消費されていきます。
何度も「やっぱりこうして」「やっぱりこう変えて」とやり直していると、トークンを無駄に使ってしまって、使用制限に引っかかってしまうんです。
制限に達すると、リセットされるまで使えなくなってしまうので、ここはぜひ意識しておいてください。

だからこそ、いきなりコードを書かせるのではなく、まずは『設計図』を作らせる。

これを「要件定義」と呼びます。

設計図を作るには、入力欄の右下「Ask before edits」をクリックして、「Plan mode」を選択します。

Plan mode なら、必ず最初に設計図を出してくれるので、いきなりコードを書き始めてしまうことを防げます。

Plan mode にしたら、いよいよ指示を出していきます。

でも、ここで多くの初心者の方が止まってしまうんです。

「そもそも、どんな指示を出せばいいかわからない…」

これ、めっちゃわかります。

僕も最初はそうでした。

Web サイトを作ったことがないのに、何を指示すればいいかなんてわかるわけないんですよね。

(以下の説明でスライドを表示)

ちなみに、こうした AI への指示文のことを「プロンプト」と呼びます。

どれだけ良いプロンプトを書けるかで、AI から引き出せる結果のクオリティが大きく変わるんです。

これ、本当にバカにできなくて、

プロンプトの質 = 成果物の質

と言ってもいいくらいです。

そこで、僕が普段から使っている、とっておきのテクニックを教えます。

それが、「AI に質問してもらう」という発想です。

自分から「あれを作って」と要望を伝えるのではなく、AI に「どんなものが作りたいですか?」と質問してもらい、自分はそれに答えるだけ——という状況を作るんです。

具体的な指示内容は、こうです。

━━━━━━━━━━━━━━━━━
【プロンプト】

僕はプログラミング初学者で、Web サイトを作ったことがありません。カフェの Web サイトを作りたいのですが、専門用語を使わずに解説しながら、一つ一つ質問をして『どのようなサイトにしたいか』を僕から引き出してください
━━━━━━━━━━━━━━━━━

こう指示するだけで、あとは Claude Code が上手にリードしてくれます。

ここから先の質問はユーザーや状況によって異なるので、画面に出てくる選択肢を一つずつ選んでいけば OK です。

例えば僕は、「作ろうとしているカフェは、実在するお店ですか?」と聞かれて、「完全に架空のお店」を選択。あとは同じように、選択肢を選んで「Submit answers」を押す流れを繰り返していきます。

質問もすっごく丁寧で、親身に話を聞いてくれるお店の店員さんみたいですね。

ちなみに、選択肢に「Other」が用意されている質問もあります。これを選ぶと自分で自由に入力できるので、僕は店名を「BLUE HOUR COFFEE」と入力しました。

ブルーアワーって、夕方の薄暗くなる時間帯のことなんですけど、ちょっとおしゃれっぽくないですか?

自分で考えながら、ちょっとワクワクしました。

こうして全ての質問に答え終わると、画面の右側に作成された計画書が表示されます。

そして、「Accept this plan?(この計画を承認しますか?)」と聞かれます。

ここでは、いったん計画づくりで止めたいので、「No, keep planning」を選びましょう。

こうして、計画書ができあがりました。

いやもう、見てくださいこれ。サイトの全体像、使う技術、ファイル構成、デザイン指針まで、しっかり整理されているんですよ。

僕、初めてこれ見たとき、「え、ここまでやってくれるの?」って本気で驚きました。

普通、これを作るのにでかいプロジェクトだと 1 週間くらいかかるのが、質問に答えただけで数分でできてる。もう本当に時代が変わりましたよね!

■ 本編 5:計画書の修正
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

計画書ができて、「お、これですぐ作れそう!」と思った方、ちょっと待ってください。

ここで、もう一つ大事なコツがあります。

それは、計画書をそのまま受け入れないということ。

計画書はあくまで Claude Code が考えた「たたき台」。

ここから、自分の意図を加えて磨き込んでいくことで、本当にイメージ通りのサイトができあがります。

ちなみに、計画書の内容はユーザーごとに違います。

なので、これからお見せするのは、僕の計画書を見て気になった部分の修正例です。

皆さんも自分の計画書を見て、「もっとこうしたい」「ここが気になる」と思う部分があれば、同じやり方で修正してみてくださいね。

例えば、僕がこの計画書を見て気になったのは「メニュー」のところが、ちょっとざっくりしすぎていることでした。

計画書には「ドリンク 3〜6 種、フード 2〜4 種」と書かれているだけ。これだと、Claude Code が実際にコードを書くときに、何種類作るか迷ってしまうんです。

なので、もっと具体的に指示を出していきます。

入力欄に、こう書きます。

━━━━━━━━━━━━━━━━━
【プロンプト】

Menu セクションは、以下のように修正してください。
・Coffee 4 種
・Main Menu 6 種
・Side Menu 2 種
このカテゴリ分けで、合計 12 種類のメニューを作ってください。
━━━━━━━━━━━━━━━━━

こう指示するだけで、Claude Code が計画書を更新してくれます。

はい、計画書が更新されました。

見てみると、Menu セクションが 3 つのカテゴリに整理されて、それぞれに具体的なメニュー名まで提案されていますね。

どうですか?

さっきの計画書より、何を作るのか一気にハッキリしたのがわかりますよね。

ちなみに、こうやって指示を出すときに「数字」を入れるのは、ちょっとしたコツです。

「メニューを充実させて」より、「Coffee 4 種、Main Menu 6 種、Side Menu 2 種」のほうが、Claude Code も迷わず動けるんです。

計画書を磨くときに、具体的な数字で指示できないかな?って意識してみると、より思い通りのものができますよ。

このように、計画書をチェックして、問題がある部分を修正して、より良いものにしていく。

こうすることで、Claude Code が最初に作る Web サイトが、こちらの理想に近いものになるんです。

計画書をしっかり磨いた後なので、ここからは Claude Code に任せて大丈夫です。

作業の途中で、こんな確認画面が出てくることがあります。

これは、「このフォルダにファイルを作っていいですか?」という許可を求められているんです。

Claude Code は、勝手にファイルを作らないように、最初の作業前に確認してくれます。

これは安全のための仕組みなので、「Allow(許可する)」をクリックして進めて OK です。

(3 分ほど経ちました)

ファイルが作成されました!

左側のファイルのアイコンをクリックすると、「cafe-site」フォルダの中に、index.html や css フォルダ、js フォルダなど、計画書通りのファイルが作られているのが確認できます。

では、実際に Web サイトを開いてみましょう。

「index.html」をクリックしても、プレビューが表示されないですよね。

こういう時は、Claude Code に開いてもらいましょう。

このように、人間にお願いするのと同じように、言葉で伝えるだけでいいんです。

すると——

ほら、Claude Code が「原因を確認します。CSS の設定を修正しますね」と返事をして、勝手にコードを書き換えてくれました。

ブラウザを更新してみると——

完璧です! 画像が綺麗に枠内に収まりましたね。

このように、エラーや不具合が出たとしても、「どこがおかしいか」を言葉で伝えて、AI に直させる。

プログラミングを始めた頃の僕なら、こういうの一つで 2 時間溶かしてました。それが今は、「はみ出してるよ」って言うだけで、勝手に直る。この体験を知ったら、もう昔には戻れないですよ、本当に。

Claude Code に任せれば、コードがわからなくても、エラーを直すことができるんです。
