# 参考動画 1: 改善点

- URL: https://youtu.be/dvk3rm24mrk?si=nC5cXJ0aSWzlingO
- タイトル: 【激震】外注なら30万のWebサイト、Claude Design × GPT image 2で作ってみた！

---

## 1. フィラーワード（「ま、」「え、」）が多すぎる

文字起こしを見ると「ま、」「え、」「えっと、」が1文に複数回出てくる箇所が大量にある。台本がない or 台本があっても読み流していると推測される。視聴者に「準備不足」「自信がなさそう」という印象を与え、テンポも悪くなる。**台本を1文ごとに改行して書き、フィラーを排除した状態で読む** だけで完成度が大きく上がる。

## 2. ED（締め）が「いかがだったでしょうか?」から始まっている

「いかがだったでしょうか?」は **動画終了を匂わせる定型句** で、視聴者が「あ、もう終わりか」と感じて離脱するトリガーになる。本編の最終トピック（セクション追加・レスポンシブ完成）から地続きで「ただ、この手法には一つ落とし穴があって〜」のように、**本編の論理を延長する形** で締めに入った方が CTA まで聞いてもらえる。

## 3. ED でのまとめが冗長

「素材が大事」「ベストプラクティスは自分で構築」「スキルも自分で育てる」と、本編で既に語った内容を ED でもう一度繰り返している。視聴者は **「動画のまとめ」が始まると離脱する** ため、ED は CTA とプレゼント訴求に絞った方がよい。思想の話は本編の中で1回だけ語る方が刺さる。

## 4. CTA（プレゼント訴求）の論理接続が弱い

中間 CTA（Claude Code アカデミー）も ED（Next.js 書籍 + アカデミー）も、**本編の Claude Design × GPT image 2 ワークフローと直接の繋がりがない**。視聴者が「今、それを受け取らなきゃ」と思える論理が不足している。例えば「この素材生成プロンプトのテンプレ集」「Claude Design 専用プロンプトテンプレ」「LP制作の素材ライブラリ」など、**今日の本編で扱った内容の延長線上にあるプレゼント** を用意した方が刺さる。

## 5. CTA のキーワード（合言葉）が無い

LINE登録 → キーワード送信 → 特典受け取り、という確実な動線が組まれていない。Claude Code アカデミーは「概要欄から購入」となっており、**無料で受け取れる入口がない** ため、購買意欲が低い視聴者は即離脱する。LINE で5文字以内のキーワード（例：「素材」「LP神」など）を送ると素材プロンプト集が届く、のような **無料の受け取り口** を作るとリスト化につながる。

## 6. 「2、3時間」の根拠が動画内で完結していない

冒頭で「2、3時間で作った」と謳っているが、動画内では工程の一部だけを見せて「裏で修正しました」「裏で微調整しました」とジャンプしている箇所が複数ある。**実際に何分かかったかのテロップ表示**（「ここまで30分経過」など）を入れると、主張の信頼性が一段上がる。

## 7. 工程⑦（Claude で素材を分割・透過）の説明が薄い

「クロードに渡して1個1個分割してと言えばやってくれます」と一言で済ませているが、**ここが手法の心臓部**。具体的にどんなプロンプトを投げて、どんなプログラムが生成され、どう実行したのかをもっと丁寧に見せた方がよい。視聴者が真似する時に一番詰まるポイント。

## 8. 構造化プロンプトの全文が画面に映っていない

「私が事前に用意しているプロンプト」と紹介しているのに、画面に全文が表示されないまま流れる。**プロンプト全文をテロップで表示** するか、概要欄に貼るかのどちらかが必要。視聴者は手を止めて読みたい情報のはず。

## 9. タイトルの「激震」「30万」のインパクトに対し、サムネと冒頭ベネフィットが地味

タイトルは「外注なら30万」と数字で煽っているのに、本編は淡々と画面操作を見せている。**「外注なら30万、AIなら数百円」のコスト比較**、**「3時間でこれが作れる衝撃」** などをもっと感情を込めて1回煽っておくと、タイトルとのギャップが減る。

## 10. 中間 CTA への入りが唐突

「じゃ、ちょっと待ってる間に宣伝なんですけど」と完全に宣伝モードへ切り替わる。**直前のトピック（Claude Code への移行）から自然に橋渡し** する設計にした方が離脱が減る。例：「Claude Code でここまでできるようになるには、最初のセットアップが結構難しいんですよね。実は僕、その入門講座を作っていて〜」のような繋ぎ。

## 11. セクション間の繋ぎ言葉が単調

「はい、ということで」「ま、こんな感じで」が多用されている。**接続詞の引き出し** が少ないため、視聴者の耳に同じパターンが繰り返される。「で、」「あと、」「ただ、」「さらに、」「しかも、」「ここで重要なのが、」などを意識的に散らした方がテンポが出る。

## 12. 「画質が荒い」を3回くらい自分で言ってしまっている

GPT image 2 の出力解像度が低いことを動画内で何度も言及している。**ネガティブ情報は1回だけ触れて、解決策（「もう少し大きい画像で出力して投化すればOK」）に即繋ぐ** べき。何度も「画質が荒いんですけど」と繰り返すと、視聴者に「結局 微妙なクオリティの動画なのか」という印象を残してしまう。

## 13. 視聴者の代弁・感情の動きが少ない

「『そんなの簡単にできるわけないでしょ?』そう思いますよね?」「『画像生成こんなに進化してたの?』って驚きませんでした?」のような **視聴者の感情を代弁する一言** がほぼない。淡々と「こうやります」「次にこうします」が続くため、感情的な惹き込みが弱い。

## 14. ナンバリング・トピックコールが弱い

本編は「次にこれ」「次にこれ」と続くだけで、**「STEP1: モック画像生成」「STEP2: 素材抽出」** のようなナンバリング・テロップが無い。視聴者が「今どの工程にいるか」を見失いやすい。テロップで明示すると視聴維持率が上がる。

## 15. デプロイのくだりがあっさりしすぎ

「デプロイも全部 Claude Code がやってくれました」で済ませているが、**ここも視聴者が知りたい部分**。Vercel か Cloudflare か、どんな指示を投げたのかを 30 秒だけでも見せると価値が増す。

## 16. レスポンシブ確認シーンが短い

「スマホサイズでもこんな感じで見れます、ハンバーガーメニューも付いてます」で 5 秒くらいで終わってしまう。完成品の **PC表示とスマホ表示の対比**、**ハンバーガーメニューを開いた状態** など、もう少し丁寧に見せれば「ちゃんと実務レベルだ」という印象が強まる。
