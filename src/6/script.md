しかも、この人気は、一時的なものじゃないんです。

プログラミングの世界って、技術の流行り廃りが激しくて、せっかく覚えた言語が数年で「もう使わないな」となることも、正直あります。

ライブラリとは、よく使う機能をあらかじめ用意してくれている "部品の詰め合わせ" のことです。
フレームワークは、アプリを作るための "土台と骨組み" がひととおりそろっているもので、これを使うことで、本格的なアプリを速く、効率よく作れるんです。

icon-react.png, icon-nextjs.png を表示

そして、現場でいちばん使われているのが、React と Next.js です。

React は、画面を "部品" を組み合わせるように作れるライブラリで、一度作った部品を使い回せるので、開発スピードが一気に上がります。

Instagram や Facebook も、この React で作られています。
しかも、Claude を開発している Anthropic 公式も、Web アプリの開発に React を採用しているほどで、AI 時代においても最も信頼性の高い選択肢です。

Next.js は、その React をベースにしたフレームワークで、より本格的なアプリを効率よく開発できます。
React だけだと自分で用意しないといけない、ページの表示を速くする「パフォーマンス最適化」や、検索エンジンに見つけてもらいやすくする「SEO 対策」などが、最初から一式そろっているんです。
だから、React の良さはそのままに、本番で通用するレベルのアプリを、より速く形にできるんですよね。
TikTok、Netflix、OpenAI など、世界的なサービスも採用しているほど、信頼性の高い選択肢です。

Web アプリを作っていくなら、絶対に学ぶことをおすすめします。

実際、僕が AI に JavaScript を書かせると、的外れなコードはまず出てこないし、エラーが出ても、すぐ直せるレベルなんです。

初心者が AI を相棒にして学んでいくのに、これ以上ない言語なんですよ。

icon-ts..png を表示

続いて、2 つ目が TypeScript。

難易度がひとつ上がって、星 2 つ。

すでに JavaScript を学んだ方は、

「また、別の言語を、勉強しないといけないのか…大変だな…」

そう思うの、めちゃくちゃわかります。

でも、安心してください。

ひとことで言うと、TypeScript は、JavaScript の "進化版" です。

元々の JavaScript の書きやすさはそのままに、ぐっとパワーアップさせた言語、という感じなんです。

だから、JavaScript がわかっていれば、新しく覚えることは、ほんのわずか。

じゃあ、何が "進化" したのか。そして、なぜ難易度が、星 1 つぶん上がるのか。

その答えが、「型」という考え方が、加わったことなんです。

文法が難しくなるわけじゃなく、JavaScript に "型" がひとつ乗っかるだけ。だから星 2 つなんですね。

slide-5.png を表示

TypeScript では、データを入れる "箱" に「ここには数字だけ」と、先にラベルを貼っておけるんです。こういう言語を「静的型付け言語」と言います。

一方、元々の JavaScript は、その箱にラベルを貼らなくても、書けてしまう言語でした。これを「動的型付け言語」と言うんです。
じゃあ、先にラベルを貼っておくと、何がうれしいのか。

slide-6.png を表示

たとえば、"年齢" を入れる箱は、数字を入れる場所ですよね。そこに、うっかり「たろう」みたいな文字を入れてしまった、としますよね。

ふつうの JavaScript だと、こういう間違いに、実際にプログラムを動かしてみるまで、気づけないんです。

2〜3 行のコードなら、目で見てすぐ気づけるかもしれません。
でも、実際に動いているアプリのコードは、数万行にもなります。
そうなると、こういうミスは、なかなか自分では気づけないんです。

honpen-2-2.mov

でも TypeScript なら、その "年齢" の箱に、「ここには数字しか入りませんよ」と、先に宣言しておけるんです。

そうすると、もし「たろう」みたいな文字を入れようとしても、まだコードを書いている段階で、「ここ、おかしいですよ」って、すぐに教えてくれる。

「それ、めちゃくちゃ助かるじゃん」

そうなんです。

バグを、世に出る前に、未然に防げる。

僕も昔、JavaScript でアプリを作っていた頃は、エラーが出まくって、その原因を探すだけで、数時間まるごと溶けてしまう、なんてことが、ザラにありました。

でも、TypeScript に変えてからは、そのエラーが、出る前にわかるので、あの "数時間溶ける" みたいなことが、ほとんどなくなったんです。

正直、もっと早く使っておけばよかったって、思いましたね。

こうやって、ミスを早い段階で見つけられる静的型付け言語は、動的型付けにくらべて、バグが出づらく、あとから直したり機能を足したりするのもラクなんです。つまり、保守がしやすい。

だから今、開発の現場では、この静的型付けが完全にトレンドになっていて、動的型付けだった言語にも、あとから型を足せる仕組みがどんどん追加されているくらいなんですよ。

そんな流れもあって、ちゃんとしたサービスを作るモダンな現場では、今やもう TypeScript が当たり前になっているんです。

実際、世界中で使われている有名な Web サービスの多くは、今や TypeScript で作られているんです。

さっき紹介した React や Next.js も、今では JavaScript ではなく、TypeScript で書くのがすっかり主流になっています。

しかも、これは大企業だけの話じゃないんです。

これから新しくアプリを作るという企業でも、最初から TypeScript を選ぶところが、本当に多いです。

僕のところに舞い込んでくる案件も、いまは「JavaScript」ではなく、「TypeScript」というものが、すっかり多くなってきました。

(スライドを表示)

数字で見ても、その人気は本物です。

evidence-1.png を表示

さっきの Stack Overflow の同じ調査では、TypeScript は、世界で最も使われている言語の、堂々の第 6 位で 43%。

世界中のエンジニアの、4 割以上が使っている、という結果なんです。

しかも、その調査で、最も使われている言語の第 1 位に輝いていたのは、JavaScript でしたよね。

TypeScript は、その JavaScript の "完全上位互換"。

つまり、TypeScript を身につけるというのは、世界でいちばん使われている JavaScript を、まるごと自分のものにしたうえで、さらに "安全に書く力" まで上乗せできる、ということなんですよ。

そして、TypeScript も、AI との相性が抜群なんですよ。

さっき、型は、箱に貼る "ラベル" のようなもの、とお話ししましたよね。実は、このラベルこそ、AI にとって、最高のヒントになるんです。

slide-7.png を表示

というのも、AI がコードを書くときって、「ここには、何を入れればいいんだろう?」と、考えながら進んでいるんですね。

そこに、「ここは数字だけ」という型が、ちゃんと決まっていれば、AI は迷いません。ぴったり合った、正確なコードを、すぐに書いてくれる。

逆に、型がないと、AI も中身を推測するしかなくて、たまに取り違える。だから、ミスが増えちゃうんです。

僕が AI と組んで開発するときも、TypeScript だと提案の精度が一段上がるのを、毎日のように感じています。

AI を相棒にする時代だからこそ、型のある TypeScript の価値は、これからますます上がっていくんですよ。

JavaScript で "動かす楽しさ" を知って、TypeScript で "安全に作る力" を身につける。

この 2 つは、もう完全にセットだと思ってくださいね。

■ 本編 3: 難易度 ★★★ 高単価を狙うなら Go
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

icon-go.png を表示

そして、最後の 3 つ目は Go。

難易度は、いちばん高い 星 3 つ。

ただ、難しいぶん、プロの現場で本当に頼りにされる、本格派の言語です。

Go は、2009 年に、あの Google が作った、比較的新しい言語なんです。

slide-8.png を表示

さっきの JavaScript や TypeScript が「画面側」を作る言語だったのに対して、Go が得意なのは、サービスの "裏側" です。

slide-9.png を表示

たとえば、ネットショッピングで、カートに入れた商品が、あとから開いても、ちゃんと残っている。

過去に買ったものの履歴も、いつでも見られる。

これは、そうした情報が「データベース」という保管庫に、しまわれているからなんです。

そこに「保存して」「取り出して」と指示を出し、やり取りを取り仕切っている、この"裏側"の部分が、「バックエンド」です。

そして、バックエンドが作れる言語は、ほかにもたくさんあります。

そのなかで、Go が大規模なサービスでこれだけ選ばれている、一番の理由。

それは、Go で作ったプログラムが、とにかく "軽くて、一瞬で立ち上がる" ことなんです。

slide-10.png を表示

というのも、いまの大きなサービスって、ひとつの巨大なプログラムでまとめて作るんじゃなくて、機能ごとに小さく分けて、たくさん組み合わせて作るのが主流なんです。この作り方を「マイクロサービス」と言います。

小さなプログラムをいくつも動かすので、その一つひとつが軽くて、サッと立ち上がる Go は、まさにうってつけ、というわけなんですよ。

その証拠に、X や Netflix といった、みなさんも使ったことがあるような大規模サービスでも、その裏側で Go が活躍しているんです。

つまり、毎日とんでもない数のアクセスをさばく、本気の大規模サービスで選ばれている言語、ということなんですね。

それに Go は、こうした大規模なものだけじゃなく、ちょっとした小さなツールやサービスを、サッと作るのにも向いているんです。

しかも Go は、これだけ高性能なのに、言語そのものは、とてもシンプル。

ふつう、処理が速い言語ほど、文法が複雑で書くのが難しくなりがちですが、Go は逆で、余計なものがなく、迷わず書けます。

それでいて、スピードはトップクラス。この "書きやすさと速さの両立" が、プロの現場でめちゃくちゃ重宝されているんですよ。

しかも、Go も TypeScript と同じ「静的型付け言語」。だから、型のおかげでバグが出づらく、あとから直したり機能を足したりするのもラク。たくさんのアクセスをさばく大規模なサービスを、長く安全に育てていくうえで、これは本当に大きな強みなんです。

実際、僕自身、もともとは Ruby や Python でバックエンドを作っていたんですが、試しに Go で書いてみたら、あまりの快適さに、それ以来、バックエンドは Go でしか作らなくなったくらいなんです。

でも、実はこの Go、難易度だけは、3 つの中でいちばん高いんです。

「あれ、シンプルなのに、なんで難易度は星 3 つなの?」

そう思いましたよね。

slide-11.png を表示

難しさの一番の要因は、ポインタ、インターフェース、ゴルーチンといった、しっかり理解するのに骨が折れる概念を、避けて通れないところにあります。

どれも、プログラムを "速く・効率よく" 動かすための、強力な概念。でも、その一つひとつが、なかなか手ごわいんです。

さっきの JavaScript では、そもそも、こういったことを深く考えなくても、プログラムは書けていましたよね。

むしろ、初心者がつまずきやすい難しい部分は、言語のほうが、うまく避けてくれていたんです。

でも Go では、そうはいかない。

これらと正面から向き合って、使いこなさないといけない。

そのぶん、難易度が、ぐっと上がるんですね。

そして、ここでもやっぱり、AI との相性の話です。

Go は、書き方のルールがシンプルで、人によるクセやブレが少ない言語なんです。

しかも、TypeScript と同じく、ちゃんと「型」もある。

だから、AI が迷わずに、正確なコードを書きやすいんですよね。

新しい言語って、本来は情報が少なくて AI も苦手なはずなんですけど、Go はそのシンプルさのおかげで、弱点をしっかりカバーできているんです。

そして、もしこの先、エンジニアとして働きたいなら、うれしい話があるんです。

Go は、エンジニアの単価が、とにかく高いんです。

書けるエンジニアがまだまだ少なくて、それでいて、現場の需要はぐんぐん伸びている。

この「人が足りないのに、求められている」という状態なので、フリーランスの案件単価も、トップクラスなんですよ。

僕の周りでも、Go を書けるエンジニアは、本当に引っ張りだこです。

しかも、これから需要が伸びていく AI サービスや、大規模なシステムの裏側は、まさに Go が活躍する舞台なんです。

つまり、Go が書けるというだけで、代わりのきかない、市場価値の高いエンジニアになれるんですよ。

「でも、結局むずかしいんでしょ?」

そう思いましたよね。

正直に言うと、JavaScript よりは、ちょっと歯ごたえがあります。

僕自身、Go を初めて勉強したときは、本当に苦労しました。当時はまだ AI もなかったので、ひとつ理解するだけでも、とんでもなく時間がかかったのを、今でも覚えています。

でも、今は、AI という最強の相棒がいる時代なんです。

シンプルな Go なら、AI に隣で教えてもらいながら進めれば、独学でも十分に手が届きます。

JavaScript で土台を作って、TypeScript で安全に書く力をつけて、最後に Go で、大規模で本格的なサービスまで作れるようになる。

この順番こそが、AI 時代に、いちばん遠回りしない黄金ルートなんですよ。
