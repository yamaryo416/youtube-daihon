# 参考動画 1: 改善点

参考URL: （文字起こしのみ・URL未指定）

## 1. 全体が長く、冗長な「待ち時間」がそのまま残っている
- アプリ生成やペット作成の「くるくる回っている待ち時間」が何度も発生し、その間の雑談が長い。
- 「ペットがまだできない」「もう少し待つ」という展開が複数回繰り返され、視聴維持率を下げる要因になっている。
- → 待ち時間は編集でカットし、テンポよく「指示→結果」を見せるべき。

## 2. ペット機能に時間を使いすぎて本筋がぼやける
- 「文字起こし変換アプリ開発」という本題よりも、ペット作成（しかも最後まで成功せず撮影後にようやく完成）に尺を取られている。
- 本題のアプリが完成した達成感より、ペットが微妙だった印象が残ってしまう。
- → ペットは「こんな遊び心もある」と数十秒で紹介し、本題に集中させた方が満足度が高い。

## 3. 音声入力の文字起こし精度が悪い部分がそのまま残っている
- 「今ニュです」「温泉入力（音声入力）」など、誤変換が多数そのまま字幕・音声に残っている。
- 視聴者が内容を追いにくく、プロっぽさも損なわれる。
- → テロップ補正や言い直しで、重要キーワードは正確に伝えるべき。

## 4. ターゲットが「Claude Code 経験者」に偏っている
- 「Claude Code 使ったことある人からすると」という前提が多く、完全初心者には置いていかれる部分がある。
- 「非エンジニアでも使える」と謳っているのに、説明は経験者向けの比較が中心。
- → 「初めての人はここだけ見ればOK」という初心者向けの導線を明確に分けると親切。

## 5. 結論（で、結局どっちを使うべき？）が曖昧なまま終わる
- 「どっちでもいい」「好きにしたらいい」という回答が多く、視聴者が次の行動を決めにくい。
- → 「こういう人は Codex、こういう人は Claude Code」と、視聴者のタイプ別に明確な指針を示すと行動につながる。

## 6. CTA（LINE登録）の訴求が弱く・唐突
- 動画の最後にコメント返信のついでのような形で LINE 登録の話が出るが、「なぜ今登録すべきか」の理由付けが弱い。
- 「特典10個」と言いながら中身が具体的に伝わらず、受け取る動機が弱い。
- → 本編内容と地続きの「この特典があれば今日の内容をすぐ実践できる」という論理接続が必要。

## 7. アプリ開発のゴール（完成形）が冒頭で示されない
- 「文字起こし変換アプリを作る」と決まるまでに時間がかかり、視聴者が「何を作る動画か」を掴むのが遅い。
- → 冒頭で「今日はこんなアプリが完成します」と完成形を一瞬見せると、最後まで見る動機になる。

## 8. エラー対応が行き当たりばったりに見える
- API キー設定後の不具合を「再起動したら直った」で済ませており、原因の言語化が弱い。
- → 「こういう時はまず再起動・ログ確認」という再現性のあるトラブル対処法として整理すると価値が上がる。
