# 参考動画 2: 改善点

## 1. 「チャンネル史上最長」になるほど網羅しすぎている
- 全機能を5カテゴリーで網羅した結果、動画が非常に長くなり、完走できる視聴者が限られる。
- 「気になるところだけ」と逃げ道を作ってはいるが、結局リファレンス止まりで「最後まで見た満足感」は得にくい。
- → 「初心者がまず使う最重要機能トップ5」と「上級者向け」を分け、メイン動画は短く絞った方が完走率は上がる。

## 2. 機能の「列挙」に寄っていて、ストーリー（流れ）が弱い
- 「次は〜、次は〜」と機能を順番に説明する構成で、カタログ的になりがち。
- 「この機能を使うとこんな未来が手に入る」という感情やベネフィットの訴求が淡白。
- → 1つの具体的なゴール（例: ある業務を丸ごと自動化する）に向かって機能を組み合わせて見せると、流れと必然性が生まれる。

## 3. 自己開示・実体験のエピソードが控えめ
- 「私の体感では」という言及はあるが、具体的な失敗談・感動エピソードが少なく、淡々としている。
- 説明は的確だが、語り手の人間味・熱量が伝わりにくく、記憶に残りにくい。
- → 「これを初めて使った時、声が出るほど驚いた」など、感情を伴う実体験を要所に入れると印象に残る。

## 4. ターゲットが「中級者以上」に寄っている
- Git・GitHub・ターミナル・ブランチ・ワークツリーなど、開発者前提の用語が多い。
- 「万人が使える」と締めているが、実際の説明の重心は開発経験者向け。
- → 非エンジニア視聴者向けに「ここだけ押さえればOK」の最短ルートを別立てで示すと親切。

## 5. 「で、結局あなたは何を作れるの？」の具体例が散発的
- スライド・動画・ニュースレターなど例は出るが、視聴者自身の日常業務に引きつけたユースケースが弱い。
- → 「営業職ならこう」「ブロガーならこう」と職種別の具体例があると、自分ごと化しやすい。

## 6. CTA（チャンネル登録・高評価）が定型的で弱い
- 最後の「ためになっていたらチャンネル登録や高評価を」が定型句で、登録する理由付けがない。
- 特典やLINEなど、次のアクションへつなぐ導線がなく、視聴者との関係構築が一回きりで終わる。
- → 「この内容を実践するためのテンプレ・チートシートを配布」など、本編と地続きの受け取る理由を作ると効果的。

## 7. 完成物の「ビフォーアフター」のインパクトが薄い
- 各実演は「できました」で淡々と終わり、成果物のすごさを強調する演出が弱い。
- → 「手作業なら何時間かかる作業が数分で終わった」など、時間・労力の対比を数字で見せると価値が際立つ。
