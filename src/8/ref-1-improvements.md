# 参考動画 1: 改善点

## 改善点

1. 冒頭のテーマ提示をもう少し短く強くできる

動画の冒頭は「スーパーパワーズは開発者の知見を、インペッカブルはデザイナーの知見を入れ込んでいる」という強い表現があり、惹きはあります。

ただ、その後の会話が少しゆっくり始まるため、最初の 15 秒で「なぜ今見るべきか」をもう一段はっきり出すと、視聴者の離脱を防ぎやすくなります。

例えば「AI コーディングで伸び悩む原因は、モデルではなく使わせる知識の不足です」のように、少し常識を壊す一言があると強くなります。

2. 目線の対象をもう少し明確にできる

動画内では初心者にも分かるように話していますが、実際には Claude Code、スキル、MCP、プラグイン、CLI など専門用語が多く出てきます。

初心者向けなのか、すでに AI コーディングを触っている中級者向けなのかを早めに明確にすると、視聴者が「自分のための動画だ」と思いやすくなります。

CLI とは、ターミナルに文字で命令を打って操作する方法のことです。

3. 用語説明をその場でもう少し挟むと分かりやすい

プラグイン、スキル、MCP の違いは説明されていますが、初心者には少し情報が一気に来ます。

「スキルは 1 つの作業手順、プラグインはそれをまとめた箱、MCP は外部サービスとつなぐ橋です」のように、比喩で短く整理すると理解しやすくなります。

MCP とは、AI が外部サービスやツールとつながるための仕組みです。

4. 4 つのおすすめスキルを先に一覧で見せると迷いにくい

動画では途中で 4 つのスキルが出てきますが、視聴者は情報量が多くなると「今どこの話だっけ」となりやすいです。

冒頭では目次を細かく読み上げない方が良いですが、スライド上だけで「開発なら Superpowers、全体整備なら Everything Claude Code、デザインなら Impeccable、管理なら Find Skills」と見せると整理されます。

5. Superpowers のデモが長いため、途中で得られる成果を明示したい

Superpowers のデモは具体的で良い一方、壁打ちのやり取りが長く、視聴者によっては途中で目的を見失う可能性があります。

「今やっているのは、作りたいものを設計書に変える作業です」と途中で一言入れると、初心者でもついていきやすくなります。

設計書とは、何をどう作るかをまとめた開発前の説明書のようなものです。

6. 実際に完成した成果物をもう少し見せると説得力が増す

Superpowers の説明では、設計書や実装計画ができる流れは分かります。

さらに、その設計書をもとにどんなアプリや画面ができたのかを少し見せると、「このスキルを使うと最終的にここまで行ける」と伝わりやすくなります。

7. Everything Claude Code の説明が相対的に薄い

Superpowers と Impeccable はデモがあり印象に残りますが、Everything Claude Code は概要説明に近く、少し記憶に残りにくいです。

「初心者が Claude Code を使うときの環境セット」「使い方の型をまとめて入れるもの」のように、誰がいつ使うべきかを具体化すると良くなります。

8. Find Skills の説明で少し迷いが見える

Find Skills について、「自分のスキルだけを検索するのか、公開スキルも検索できるのか」と話者自身が途中で確認しています。

正直さは良いですが、視聴者には少し不安が残るため、事前に用途を確認してから「これはおまけ枠」と割り切って紹介すると分かりやすくなります。

9. セキュリティ注意点に具体例があるとさらに役立つ

第三者製スキルには危ないものがあるかもしれないという注意は良いです。

ただ、初心者は何を見れば危ないのか分かりにくいため、「提供元を見る」「インストール数を見る」「README を読む」「権限が強すぎないか見る」のようなチェックリストを出すと行動しやすくなります。

README とは、ツールの説明や使い方が書かれた案内文のことです。

10. 中間 CTA の特典内容をもう少し具体化できる

資料を配布する流れは自然ですが、「資料」と「フォルダ」が何なのかが少し曖昧です。

「今日紹介した 4 つのスキル比較表」「インストール手順」「用途別おすすめ一覧」のように、受け取るものを具体的に言うと登録理由が強くなります。

11. LINE 登録の導線に全員対象の言い方を入れるとさらに強い

参考動画では会話の流れで自然に LINE 特典に入っています。

ただ、「欲しい方は」のような選択肢型に寄ると離脱しやすいため、自分たちの台本では「ここで全員概要欄のリンクから」と全員対象の言い方にすると良いです。

CTA とは、視聴者に登録や資料受け取りなどの行動を促す部分のことです。

12. 最後の締めをもう少し未来につなげられる

参考動画の最後は、4 つのスキルを整理して終わっています。

自分たちの台本では、さらに「スキルを使いこなせると、AI に専門家の判断を入れて、自分だけの開発環境を作れる」という未来まで広げると、エンディングの熱量が上がります。

13. チャンネル登録や次回動画への流れを強められる

最後に「今後の動画でスキル自体を解説する」と触れている点は良いです。

ただ、視聴者が次に何を見ればよいかが少し弱いため、「次回はスキルを自作して、自分の知見を AI に入れる方法を解説します」のように具体化すると、次回視聴につながりやすくなります。

14. 自分の体験や感情をもう少し強く入れられる

話者が実際に Superpowers を使っていることは伝わります。

さらに「僕も最初、要件定義を飛ばして何度も手戻りしました」「これを使った時、AI が本当に先輩エンジニアみたいに質問してきて驚きました」のような感情が入ると、視聴者に刺さりやすくなります。

15. 初心者が今すぐやる 1 アクションを提示できる

動画では多くのスキルが紹介されますが、初心者は最後に「結局何からやればいいのか」で迷う可能性があります。

「まずは skills.sh で Superpowers と Impeccable を調べて、README を読むところから始めてください」のような小さな行動を置くと、視聴後の満足度が上がります。
