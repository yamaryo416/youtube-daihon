まず最初に、Web アプリがどういう仕組みで動いているかを、ざっくり整理しておきます。

Web アプリは、大きく 3 つの部分に分かれています。

まず「フロントエンド」。
これは、皆さんがブラウザで見ている画面のことです。
ボタンを押したり、フォームに文字を入力したり、情報が表示されたり、ユーザーが直接触れる部分すべてがフロントエンドです。

次に「バックエンド」。
フロントエンドから受け取った情報を処理して、データを保存したり、別の情報を返したりする、裏側の仕組みのことです。
レストランに例えると、フロントエンドが「ホールスタッフ」で、バックエンドが「キッチン」です。
お客さんの注文を受けて、実際に料理を作るのがバックエンドの役割ですね。

そして「データベース」。
バックエンドが処理した情報を保存しておく「倉庫」のことです。
ユーザーが投稿したメモや、アカウント情報などが、ここに蓄積されていきます。

(スライドを表示)

そして、アプリを本格運用していくには、もう 2 つだけ知っておきたい概念があります。
「GitHub」と「デプロイ」です。

GitHub は、作ったコードを保存・管理しておく場所のことです。
「コードの保管場所」と覚えておけば十分です。

そして、デプロイ。
完成したアプリをインターネット上に公開する作業のことです。
「ネットに上げる」とイメージしてもらえれば OK です。

(スライドを表示)

AI が発達する前までは、もし自分でアプリを作って公開しようとすると、これらの作業をすべて一人でこなす必要がありました。
僕が独学でアプリを作っていた頃も、半年ほどかかって、本当に大変だった記憶があります。

それが今は、Claude Code によって劇的に変わりました。
フロントエンドの実装から、データベースの設計・構築、デプロイまで、この全工程を Claude Code がやってくれます。

その中のひとつが、Playwright Plugin です。

Playwright とは、ブラウザを自動で操作できるツールのことで、「このボタンをクリックして、次の画面に遷移する」といった操作を自動的に再現してくれます。

この Playwright Plugin を Claude Code に追加すると、Claude Code が自分でブラウザを開いて、画面を確認しながら動作確認まで行ってくれるようになります。
「ここにボタンがある」「この画面に遷移できる」という確認を、Claude Code が自律的にやってくれるので、開発のスピードがさらに上がります。
