# 参考動画 4: 改善点

# 参考動画 URL: なし（文字起こしのみ）

## 1. 冒頭の自己紹介・宣伝が多く、フックが弱い

「このチャンネルではエンジニアとしての幅広い知識と AI コーディングの知識を解説しているチャンネルになりますので、気になる方は今すぐチャンネル登録よろしくお願いします」という
チャンネル登録の誘導が冒頭に入っており、まだ視聴者に何も価値を提供していない段階でのCTAになっている。
「37分で課金機能付きWebサービスが完成しました。今日はその全工程をリアルタイムでお見せします」という宣言を冒頭に置く方が離脱を防げる。

## 2. 書籍・Udemy・新行動キャンプの宣伝が本編中に唐突に入る

「NEXTJS の書籍出すので4月13日販売予定なので〜」という書籍の宣伝が技術説明の途中に挟まれており、
本編の流れを断ち切っている。宣伝はED・概要欄にまとめ、本編は集中して見られる構成にする方が好ましい。

## 3. 視聴者への「あなたも再現できる」メッセージがほぼない

実演形式で高速に進むため、「これを見た視聴者が同じことを試せるのか」という視点が弱い。
「Turso の MCP サーバーの設定はこちらの動画を見れば10分でできます」など、
視聴者の次のアクションへの具体的な橋渡しがあると行動転換率が大幅に上がる。

## 4. 技術用語の説明が少なく初心者が置いていかれる

Turso・Stripe MCP・AquaVoice・Next.js・プランモードなど、複数の技術ツールが説明なしに次々と登場する。
「今日の動画は中級者以上向けです」と明示するか、各ツールに一言の説明を添えるかのどちらかに振り切る必要がある。

## 5. 「待っている37分」の見せ方がもったいない

「37分37秒待ちました。待っている間別のことをすればいい」とだけ言っているが、
その37分間に「実際に他の仕事を何をしていたか」「どんなことが考えられるか」を見せると
「AI が働いている間に人間は別のことができる」というメッセージがより具体的に伝わる。

## 6. CTAが本編内容と論理的につながっていない

動画後半での LINE 誘導・書籍宣伝・Udemy 宣伝が、本編で実演した内容（Claude Code で WebService を作る）と
論理的につながっていない。「今日作ったようなサービスをあなた自身で作れるようになるために〜」という
本編との橋渡しがあるだけで、CTAの受け取られ方が大きく変わる。

## 7. 「非エンジニアにとっての意味」が伝わらない

動画全体を通じてエンジニア・バイブコーディング経験者向けの内容が中心になっており、
「コーディングをしたことがない人がこれを見てどうアクションすればいいのか」が不明確。
「エンジニアでなくてもこのフローで〇〇できます」という具体例が1つあるだけで視聴者層が広がる。
