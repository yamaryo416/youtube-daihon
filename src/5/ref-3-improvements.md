## 1. 視聴者への「あなたもできる」メッセージが弱い

現役エンジニアが実演する形式は信頼性が高い反面、「自分にはまねできない」と感じさせるリスクがある。
「エンジニアでなくてもこの流れは同じです」「初心者でもこのフロー通りにやれば再現できます」という橋渡しの言葉が随所に必要。

## 2. 冒頭のフックが薄い

「あさんのAIエージェント実装実演講座」という冒頭から始まっており、
視聴者が「なぜ今すぐこれを見なければいけないのか」を感じる前に内容が始まってしまっている。
「AIを使えばエンジニアが丸4日かかる作業が1時間で終わります。今日はその全工程をお見せします」という宣言を冒頭に置くだけで離脱が大幅に減る。

## 3. 専門用語の説明がエンジニア目線で止まっている

`--dangerously-skip-permissions` ・ CLAUDE.md ・ Cursor ・ Node.js など、
エンジニア向けの用語がほぼ説明なしに登場する場面が多い。
非エンジニアの視聴者が「自分には関係ない話だ」と感じて離脱するポイントになっている。

## 4. 全体の進捗が視聴者に伝わりにくい

ヒアリング→要件定義→設計→実装という工程が会話の流れの中に埋まっており、
「今どのフェーズにいるか」が視聴者に伝わりにくい。
画面上に「STEP 2/4: 要件定義」のようなインジケーターを表示するか、
ナレーションで定期的に「ここまでが要件定義です」と現在地を示すだけで視聴維持率が大幅に改善する。

## 5. 「普通の人が使うイメージ」の提示が不足

「録音→AI要約→要件定義」というフローは非常に革新的だが、
「これを非エンジニアのビジネスパーソンが使うとどういうシーンで役立つか」という具体例がない。
「営業の提案書作成」「会議議事録の要約」など身近な応用例を1〜2個挟むと視聴者の行動意欲が高まる。

## 6. CTAが突然で本編と断絶している

中盤に「エンジニアファーストの公式LINE」への誘導が突然挿入されており、
本編の文脈と全くつながっていない。
「丸4日が1時間になるこの仕組みをもっと体系的に学びたい方のために、〇〇を用意しました」という
本編との論理的なつながりを作るだけで、CTAの受け取られ方が大きく変わる。

## 7. エンジニア向けの話題と初心者向けの話題が混在している

Cursor と Claude Code の使い分け・Node.js の技術選定など、
かなり上級者向けのトピックが初心者向けの解説と混在している。
ターゲットを絞るか、「ここはエンジニア向けの話なので初心者の方はスキップOKです」という案内を入れると視聴者が迷子にならない。
